KAFKA

kafka是一个分布式的流数据平台，流数据平台通常具有以下特点：

  1. 通过消息队列的方式对流数据进行发布和订阅。
  2. 以高容错性的方式储存数据。
  3. 对流数据进行实时处理。

kafka常被用于在系统和应用之间实时地获取数据。
kafka包含四大API：

  1. Producer API用于将流数据发布到一个或多个主题当中。

  2. Consumer API允许应用订阅一个或者多个主题并且对流数据进行处理。

  3. Streams API主要对流数据进行处理，可将输入流转换为输出流并发布到Topic中去。

  4. Connector API ：一个与关系型数据库建立的Connector可以捕获数据库的每一次变动。

kafka核心抽象

Topics and Logs

Topic往往代表流数据的类型，包含多个log分区。一个Topic可以有多个订阅者。

每个分区由有序的不可变的log序列组成。序列中每条记录都会被分配一个不同的id，该id被称为offset。
